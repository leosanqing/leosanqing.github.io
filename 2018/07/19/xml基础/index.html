xml-简介

名词解释

xml（extensible Markup Language）可拓展标记型语言
标记型：使用标签操作，和HTML一样
可拓展：标签可以自己定义，可以写中文标签 <猫></猫>,这点区分于HTML，HTML不能自己定义标签，必须使用给定好的标签

xml 是W3C组织发布的技术，目前有两个版本，1.0,1.1。主要使用1.0，因为1.1并不能兼容1.0的东西

xml用途

显示数据

和HTML一样，xml也可以(但是这不是他的重点)

存储数据

这是他的主要功能，可以像数据库一样进行简单的存储数据

xml-应用

不同的系统间数据传输

比如 QQ消息之间的传递

表示生活中有关系的数据


配置文件

这个在编程中用的很多，因为我们使用xml配置文件，可以增强代码的可维护性，我们修改一些配置不需要再源代码中进行修改，只需要修改相应的配置文件即可。

xml-语法

xml的文档声明

1.创建一个xml文件，后缀名是.xml
2.写xml，必须要有文档声明
格式：<?xml version="1.0" encoding="utf-8"?>
必须写在第一行第一列，必须写在第一行第一列，必须写在第一行第一列，！！！！

元素

形式（2种）

1.<tag>This is a tag</tag>
2.<tag/>

格式

1.不能交叉嵌套
2.有且仅有一个父节点

命名规则

1.不能以数字、标点符号开头
2.不能以xml、XML开头
3.不能包含空格
4.严格区分大小写

属性

1.可以有多个属性
2.要使用 “”包含
3.不能重复
4.命名规则和元素相同

注释

只有一种<!--这是注释--!>

CDATA区

character data 的缩写
作用：把标签当做普通文本
格式：<![CDATA[内容]]]>
如：

处理指令

PI(Processing Instruction)
作用：用来指挥软件如何解析文件
常用：
1.<?xml version="1.0" edcoding="utf-8"?>
2.xml-stylesheet指令：
作用：指示xml文档所使用的CSS样式 XSL
<?xml-stylesheet type="text/css" href="some.css"?>
注意：对中文不起作用

约束

是什么

约束xml的书写规范的文档称为约束

为什么要使用

xml都是用户自己定义的标签，若出现小的失误，就会造成文件无法解析，出错的情况

常用的约束技术

1.xml DTD
2.xml Schema

DTD

格式

关联方式

使用内部 DTD
外部本地引用

<! DOCTPYE 根元素 SYSTEM "DTD路径">
exm:<! DOCTPYE 书架 SYSTEM "/book.dtd">

外部网路引用

<! DOCTPYE 根元素 PUBLIC "DTD文件名" "DTD文档URL">
exm:<! DOCTPYE 书架 PUBLIC "/book.dtd" "http://Java.sdfsa/....com">

DTD 定义元素

语法： <! ELEMENT 元素名称 使用规则>
规则：
（#PCDATA）指元素主题内容只能是普通文本
EMPTY： 元素主体为空 如

ANY ：主体内容为任意类型
(子元素) 元素包含的子元素

约束的4种形式

REQUIRED IMPLIED FIXED value Ddfaultvalue

定义引用实体

概念：

在DTD中定义，在XML中使用

语法

<! ENTITY 实体名称 "实体内容">

引用方式

exm: DTD中： <! ENTITY copyright "....所有"
xml中:&copyright;

XML解析方式

DOM和SAX

区别:

DOM把整个文件放入内容，然后解析；SAX采用时间驱动，边读边解析

DOM优缺点：

优点：方便实现增删查改操作
缺点：文件过大会造成内存溢出

SAX优缺点

优点：不会造成内存溢出
缺点：不能实现增删查改功能
